Programación en Ispsoft y Dopsoft del banco de prueba





La programación de la estación 10506708 está organizada en tres modos principales de operación, los cuales se seleccionan desde la HMI. Cada modo define una secuencia lógica que el PLC ejecuta junto con los módulos de prueba y las electroválvulas.

En el modo Main se utiliza exclusivamente el módulo 10510030. El operador coloca el módulo en el banco de pruebas y presiona el botón de avance bloqueo. Después inicia la validación y posteriormente el test. Si todo funciona correctamente, los LEDs verdes en el HMI se encienden indicando que los puntos de prueba son válidos. En caso contrario, aparece un LED rojo que señala una falla en las conexiones. Para regresar el sistema a su estado inicial se presiona el botón Reset. Si el operador cambia de modo o presiona el ícono de Windows, el módulo se libera automáticamente por seguridad, ya que este módulo solo puede usarse en Main.

En el modo IOS la estación permite al operador realizar pruebas manuales. El módulo se coloca y se activa el avance bloqueo, después se habilita el modo manual en la HMI. En este estado, el operador puede activar electroválvulas o LEDs de forma directa y observar su funcionamiento, lo que facilita la detección visual de fallas. Si se cambia de modo o se presiona el ícono de Windows, el módulo se libera para evitar que quede en un estado de prueba activa.

En el modo Comprobación de Módulo, el sistema identifica el binario único de cada módulo. Tras colocar el módulo y activar el avance bloqueo, se presiona el botón de validación del binario. El sistema compara el código del módulo con su base de datos y, si coincide, abre una pantalla específica en la HMI para ese módulo. Si no coincide, se muestra un error y la prueba se reinicia. Una vez validado, se presiona iniciar test e inmediatamente se ejecuta una prueba automática en la cavidad correspondiente. Si la prueba es exitosa, aparece un LED verde con el mensaje de validación; si falla, se enciende un LED rojo y se indica la falla detectada en el conector. Para repetir la prueba se utiliza el botón liberar pieza, que reinicia el proceso. Finalmente, con el botón regreso se vuelve a la pantalla inicial y se libera el módulo, garantizando que el sistema quede listo para la siguiente prueba.

En conjunto, la programación no está escrita en código visible dentro del manual, sino que se describe como secuencias de operación controladas por el PLC y la HMI. Cada modo representa un bloque lógico de pruebas: el primero asegura el funcionamiento general de la estación, el segundo permite comprobaciones manuales y el tercero identifica y valida automáticamente los módulos instalados.
